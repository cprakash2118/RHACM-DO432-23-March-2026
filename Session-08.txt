POint-01-# 
RHACM does not just "connect" cluster like ssh.

Instead it:-
- Install agent 
- Establishes secure trust 
- Maintains continuous heartbeat (Refer even NTP also - chronyd)

2016 + 1 secoud  - 2017 
MachineCOnfig Change to all nodes 


1-) RHACM is agent-based + Certificate-Based + lease-based management:-
- Network must be perfect 
- Certificates must be trusted 
- Time must be synchronized
- Agent must continuously report.

NOte: If any one fails -> Cluster status breaks.

2-) RHACM cluster lifecycle (High-Level flow):-
- HUB cluster runs "Import COntroller"
- Import secret is generated on HUB
- Managed cluster installs "klusterlet"
- Managed cluster registers with HUB.
- Managed cluster joins 
- Managed cluster sends heartbeat (lease)
- RHACM shows Ready/Available.



I create/imported a cluster, but it never shows up properly. Why like this ?
- Network connectivity issue (Most common issue)
  RHACM requires bi-directional connectivity.
   
   From        to           port 
   HUB       Managed        6443 
   Managed   HUB            6443 
   Both      Both           443 
   
   NOte: DNS must resolve API endpoints 
   What goes wrong ?
   - Firewall bloack 6443 
   - Private DNS mismatch (zone records)
   
   curl -k https://api.ocp4.example.com:6443/readyz
   curl -k https://api.ocp4-mng.example.com:6443/readyz   
   
   
- Certificate Trust issue:-
  My cluster API is reachable. But still import fails ... why ? 
  Reachability not equal to Trust.
  
  RHACM use mTLS (Mutual TLS)
  
  Problem might be because of many issue (just mentioned few here).
  - You might used custom CA (not proper)
  - HUB does not trust Managed cluster CA.
  
  $ openssl s_client -connect api.ocp4-mng.example.com:6443 
  
  Note:- If ccert chain is untrusted -> registration fails silently.
  
  FYI- RHACM is strict about certificate -> No trust = No management.
