****************** Installation Prerequisites & sizing *********************
1-# Platform Requirement:-
  - OpenShift Cluster (recommended).
  - Kubernetes is supported, but openshift gives:-
    - Integrated OperatorHub
	- Oauth 
	- Observability 
	- Lifecycle automation.
  - FYI
    - Why OpenShift is recommended?
      RHACM heavily depends on OpenShift operator, APIs etc.

2-# Browser requirment [Chrome/Safari/Edge]

3-# Network Connectivity.
    RHACM requires bi-directional connectivity between:-
     - HUB cluster 
     - Managed cluster 

 	FOr what ?
    - Agent registration 
    - Policy Enforcement
    - Metrics collection 
    - Workload Orchestration 

    Alone with this if you have disconnected cluster:-
    - Images must exist in local registries (Offline quay.io)
    - OperatorHub must be mirrored (Using offline quay.io).

4-# Sizing factor - [ Consider this point like "What actually consumes resources?"]
    RHACM resources usage depends on:-
    1-# Number Of cluster [Each cluster run agents]
    2-# Number of polciies [Each policy = reconcillation loops]
    3-# Account Acount [Drives placement & Monitoring]
    4-# Storage Speed [Imaacts Observability part]
    5-# Network Latency [AAffect agent sync process]

  RHACM is control-Plane.
  
  
  FYI- Once RHACM deployment then application running on multiple namespace (open-cluster-management).
  
4-# Why dedicated namespace matters?
   RBAC isolation 
   Easier cleanup 
   Security Audits etc 

5-# Update approval - Manual (As per Prod env).
    - Generally RHACM upgrade may cause like -> Changes policy CRDs, Affect governance, restart controller.

 Once RHACM operator installed.
- Operator pods started 
- CRDs are registered 
- Controllers are loaded 
  etc.
